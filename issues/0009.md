## Description

Surface upload conflicts (HTTP 409) to the user without losing any edits. Implementation is in the `../Issues` Mac app. This is Phase I from `RemoteSync.md`.

Conflict policy (v1):

1. Rename the local file to `NNNN.md.local-conflict-<yyyyMMdd-HHmmss>`
2. `GET` the server's current version, write to `NNNN.md`
3. Show a one-line banner in the app: "Issue 0042 changed on the server; your edit was saved as `0042.md.local-conflict-…`"
4. Banner links to a side-by-side diff sheet (small SwiftUI view, line-level diff)
5. User merges manually, edits `0042.md`, FSEvents fires, normal upload path takes over

No edit is ever silently lost. More sophisticated merge flows can come later.

## Notes

The conflict file naming pattern `NNNN.md.local-conflict-<yyyyMMdd-HHmmss>` must not trigger a new upload from FolderWatcher. The watcher should filter out `*.local-conflict-*` files or SyncEngine should suppress them explicitly.
