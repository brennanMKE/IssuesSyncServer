## Description

Add write endpoints with optimistic ETag concurrency. ETag = lowercase hex SHA-256 of file bytes, stored in `files.etag` and S3 object metadata. This is Phase D from `RemoteSync.md`.

Endpoints to implement:

| Method | Path | Auth | Body / Headers | Returns |
|---|---|---|---|---|
| `PUT` | `/v1/folders/{folderId}/issues/{id}` | editor+ | Raw markdown, `If-Match: <etag>` or `If-None-Match: *` for create | `200` + new `ETag`; `409` on mismatch; `412` if `If-Match` missing |
| `POST` | `/v1/folders/{folderId}/issues` | editor+ | `{ id, body }`; validate `^\d{4}$` and uniqueness | `201` + `ETag`; `409` if id exists |
| `DELETE` | `/v1/folders/{folderId}/issues/{id}` | editor+ | `If-Match: <etag>` | `204`; `409` on mismatch |
| `PUT` | `/v1/folders/{folderId}/issues/{id}/attachments/{name}` | editor+ | Raw bytes, `If-Match` / `If-None-Match: *` | `201` / `200` + `ETag` |
| `DELETE` | `/v1/folders/{folderId}/issues/{id}/attachments/{name}` | editor+ | — | `204` |
| `PUT` | `/v1/folders/{folderId}/project.json` | admin+ | Raw bytes, `If-Match` | `200` + `ETag` |

Also: write every mutation to `audit_log` (actor, action, project, path, etag_before, etag_after, IP, user-agent).

In-process LRU cache (~64 MB) for recently-read bytes; invalidate on `PUT`/`DELETE` for the affected key.

## Root cause

Phase D write endpoints were unimplemented — all routes returned 501.

## Fix

Added conditional `UpsertFile`/`DeleteFile` in the storage layer using atomic SQL (UPDATE WHERE etag=… / INSERT ON CONFLICT DO NOTHING). Six write handlers in `internal/api/v1/write.go` enforce role authorization, 25 MB body cap, SHA-256 ETag, S3 write, DB upsert/delete, LRU cache invalidation, and audit log write.
