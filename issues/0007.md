## Description

New "Remote sync server" section in Issues.app Settings that lets the user configure a backend URL, sign in via PassKey, and bind a local folder to a remote project. Implementation is in the `../Issues` Mac app, but the backend API surface it depends on is defined here. This is Phase G from `RemoteSync.md`.

Mac-side deliverables (in Issues repo):

- Settings UI: server URL entry field, sign-in button (triggers PassKey assertion), signed-in state with account name
- `trustMode` per server entry: `publicCA` (backend) vs `pinnedSelfSigned` (Mac-as-host, existing); `RemoteClient` switches delegates accordingly
- Project list view after sign-in (`GET /v1/folders`)
- "Connect this folder" picker: security-scoped bookmark, requires empty folder; if non-empty, show "Pick an empty folder — the app needs to download a fresh copy"
- Persist `(serverURL, projectId, etagPerFile, lastSeenEventId)` across launches

Backend API surface needed (already covered by Phases B–C):

- `POST /v1/auth/challenge` — issue WebAuthn challenge for native client
- `POST /v1/auth/assert` — verify assertion, return JWT access token + refresh token
- `POST /v1/auth/refresh` — exchange refresh token for new access + refresh tokens
- `GET /v1/folders` — already in Phase C

## Notes

The existing `RemoteClient`'s `Authorization: Bearer` mechanism is reused — only the token's content changes (JWT vs opaque session cookie on the web side).

Backend prerequisite (Phases B + C) is complete as of commits 844e19d + da902fc. Remaining work is in `../Issues` (Mac app).
