## Description

Implement the full auth stack. No passwords are stored at any point. This is Phase B from `RemoteSync.md`.

Deliverables:

- WebAuthn registration and assertion using `github.com/go-webauthn/webauthn`; RP ID = public DNS name
- Admin bootstrap from `ADMIN_EMAIL` env: create user row on first boot, mark email unverified, print one-shot enrollment URL (also available via `issued admin enroll-url`); enrollment link valid for 24 hours
- Web session: cookie (`HttpOnly; Secure; SameSite=Strict`), server-side session record in Postgres; logout/revoke is real
- Native app session: PassKey assertion → short-lived JWT access token (15 min) + opaque refresh token (30 days, rotated on use); both stored in Mac Keychain; `Authorization: Bearer` header on API calls
- Invite flow: admin enters email + role + project assignments → signed JWT invite token (7-day, single-use) → SES email with one-click link; invitee registers PassKey, no password ever set
- `401` on missing/invalid/expired token; `403` on authorization failure
- `sessions` table records for revocation; refresh token rotation with 30-second grace window for racing requests
