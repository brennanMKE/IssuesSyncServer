## Description

New `SyncEngine` module in the Issues app that sits between `FolderBookmarkService` (local folder) and the remote backend. Implementation is in the `../Issues` Mac app. This is Phase H from `RemoteSync.md`.

Deliverables:

**Initial sync (on folder bind):**
- Pause `FolderWatcher`
- Download every issue file, attachment subdirectory, and `project.json` into the chosen (empty) folder
- Resume watcher once disk state matches server; record `etagPerFile`

**Steady state — local edit detected:**
- Debounce 150 ms
- Compute SHA-256 of changed file
- If hash ≠ known ETag → `PUT … If-Match: <known-etag>` (or `POST`/`PUT … If-None-Match: *` for new files)
  - `200` → record new ETag
  - `409` → conflict path (see Phase I, issue 0009)
- If file deleted → `DELETE … If-Match: <known-etag>`

**Steady state — remote change via WS:**
- If ETag matches "expecting" record from own upload → skip (echo suppression)
- Otherwise → `GET` the file, atomic temp-write + rename, record new ETag
- For `deleted` op → remove local file

**Watcher feedback loop:**
- Files just written by SyncEngine will fire FSEvents; suppress upload if ETag matches `etagPerFile`

**Offline + reconnect:**
- WS dropped → exponential backoff reconnect (port from Mac-as-host path)
- On reconnect → `subscribe` with `since: lastSeenEventId`; handle `resyncRequired` with full reconcile
- Local edits made while offline → queue in memory + small on-disk log; replay on reconnect with conflict path

## Notes

Atomic write = write to temp file then `rename(2)` so FolderWatcher sees one event per file, not a partial write.
