## Description

Server-rendered HTML admin console at `/admin/*`. Same Go binary — no separate process. Uses `html/template` + htmx for interactive bits; no JS build pipeline. This is Phase F from `RemoteSync.md`.

Routes:

| Path | Auth | Purpose |
|---|---|---|
| `/admin` | session | Dashboard: project list, recent activity |
| `/admin/login` | — | Email entry → PassKey assertion |
| `/admin/enroll/{token}` | invite token | First-time PassKey registration |
| `/admin/users` | global admin | List, invite, change role, deactivate |
| `/admin/users/{id}` | global admin | User detail; revoke sessions; reset PassKey set |
| `/admin/projects` | global admin | List, create, archive |
| `/admin/projects/{slug}` | project admin | Edit metadata; member list |
| `/admin/projects/{slug}/members` | project admin | Add/remove members; change per-project roles |
| `/admin/projects/{slug}/files` | project admin | Read-only file browser (sanity inspection) |
| `/admin/audit` | global admin | Paginated audit log, filterable by user/project/action/time |
| `/admin/sessions` | session | Your own active sessions; revoke |
| `/admin/devices` | session | List PassKeys; remove one; start add-device QR flow |

CSS: single stylesheet, system fonts, no framework. Visual tone: neutral, readable, dense — match the Issues app.

HTML/CSS/JS embedded via `embed.FS` (set up in Phase A).

## Root cause

Phase F admin console was a stub returning 501.

## Fix

Created full `internal/admin/` package: 30 route handlers on `adminServer`, in-memory WebAuthn challenge store, `embed.FS` for 13 HTML templates and a single `style.css`. All admin routes protected by `auth.RequireWebAuth` middleware except `/admin/login` and `/admin/enroll/{token}`. Storage layer extended with 20+ admin query functions in `storage/admin_queries.go` and row types in `storage/types.go`.

## Files changed

- `internal/storage/types.go` — all admin row struct types
- `internal/storage/admin_queries.go` — 20+ Postgres query functions for admin operations
- `internal/admin/admin.go` — full adminServer with all 30 handlers, WebAuthn session store, embed.FS
- `internal/admin/static/style.css` — single ~175-line stylesheet
- `internal/admin/templates/` — 13 HTML templates (base, login, enroll, dashboard, users, user, projects, project, members, files, audit, sessions, devices)
- `internal/api/router.go` — added `AdminHandler http.Handler` to Deps; delegated `/admin/` route
- `internal/auth/webauthn.go` — added `InviteKey()` accessor
- `cmd/issued/main.go` — wired `admin.Handler` into `api.Deps`
