## Description

Port the existing read endpoints so the Mac Issues.app can point at the backend and work without modification. This is Phase C from `RemoteSync.md`. No write operations yet — the viewer behaves exactly as it does today against a Mac-as-host server.

Endpoints to implement:

| Method | Path | Notes |
|---|---|---|
| `GET` | `/v1/host` | `displayName` = configured server name; `folderCount` = projects this user can access |
| `GET` | `/v1/folders` | List of `FolderInfo` for projects the authenticated user is a member of |
| `GET` | `/v1/folders/{folderId}` | Single `FolderInfo` |
| `GET` | `/v1/folders/{folderId}/issues` | `[IssueMetadata]`; add `etag` field (nullable for old clients) |
| `GET` | `/v1/folders/{folderId}/issues/{id}` | `IssueDetail` with `ETag` response header (SHA-256 of bytes) |
| `GET` | `/v1/folders/{folderId}/issues/{id}/attachments/{name}` | Streamed bytes |

The `wire/` package types must match `RemoteProtocol.swift` field names exactly. New fields (`etag` on `IssueMetadata`) must be `omitempty` so older Mac-as-host responses still decode.

Smoke-test by pointing Settings → server URL at the backend and confirming the viewer works end-to-end.

## Root cause

Phase C endpoints were unimplemented — all `/v1/*` routes returned 501.

## Fix

Added storage query layer (`storage/queries.go`, `storage/s3read.go`) and six read-only handler files under `internal/api/v1/`. All routes wrapped in `auth.RequireAuth` middleware. Path parameters use Go 1.22+ `{name}` syntax with `r.PathValue`. ETag headers double-quoted per RFC 7232.

## Files changed

- `internal/storage/queries.go` — six Postgres query functions (projects, issues, attachments, user lookup)
- `internal/storage/s3read.go` — `GetObject`, `StreamObject`
- `internal/api/v1/deps.go` — shared `Deps` struct and `writeError` helper
- `internal/api/v1/host.go` — `GET /v1/host`
- `internal/api/v1/folders.go` — `GET /v1/folders` + `GET /v1/folders/{folderId}`
- `internal/api/v1/issues.go` — `GET /v1/folders/{folderId}/issues` + single issue endpoint
- `internal/api/v1/attachments.go` — `GET /v1/folders/{folderId}/issues/{id}/attachments/{name}`
- `internal/api/router.go` — added `RPDisplayName` to Deps; replaced `/v1/` stubs with real routes
- `internal/auth/webauthn.go` — added `JWTKey()` accessor
- `cmd/issued/main.go` — wired `RPDisplayName`, `S3`, `S3Bucket` into `api.Deps`

## Notes

A round-trip test fixture shared between the Go `wire/` package and the Swift `RemoteProtocol.swift` should be set up here to catch drift early.
