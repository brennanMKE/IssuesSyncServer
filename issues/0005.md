## Description

Implement the `/v1/events` WebSocket hub with per-folder subscription, event fan-out, replay-from-id, and `accessRevoked` on member removal. This is Phase E from `RemoteSync.md`.

Server → client message types:

```jsonc
{ "type": "subscribed", "folderId": "…" }
{ "type": "issueChanged", "folderId": "…", "issueId": "0042",
  "op": "created" | "updated" | "deleted",
  "etag": "ab12…",        // null for deleted
  "actor": "user-uuid",   // client suppresses echo of own writes
  "ts": "2026-05-18T12:34:56.789Z" }
{ "type": "attachmentChanged", … }
{ "type": "projectMetaChanged", … }
{ "type": "accessRevoked", "folderId": "…" }
```

Client → server (existing protocol, unchanged):

```jsonc
{ "type": "subscribe", "folderId": "…" }
{ "type": "unsubscribe", "folderId": "…" }
```

Extended subscribe for reconnect replay:

```jsonc
{ "type": "subscribe", "folderId": "…", "since": "<lastSeenEventId>" }
```

The server replays missed `events` rows since that id. If the gap is too large (GC'd), respond with `{ "type": "resyncRequired", "folderId": "…" }` so the client falls back to a full `GET /v1/folders/{id}/issues` reconcile.

`events` table GC: delete rows older than N hours (configurable; default 24 h).

## Notes

Apache 2 `mod_proxy_wstunnel` handles the WebSocket upgrade. `ProxyTimeout` must be set to `3600` or connections will be killed after Apache's default 60-second idle timeout. This was already noted in `0001`.
